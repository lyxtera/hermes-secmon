# Cron Notification Formatting

All secmon cron wrapper scripts (tick.sh, audit.sh, daily.sh) produce Markdown-formatted output for Telegram delivery via the Hermes Gateway.

## Header pattern

```
## 🔔 Secmon Tick
*2026-07-02 04:25 UTC*
```

H2 header with emoji + italic timestamp on the next line.

## Alert lines (from `alerts.py dispatch()`)

```
**{severity}** `{source}`: {message} → `{fix_command}`
```

- Severity: **bold** (`**HIGH**`)
- Source: backtick (`` `self_protection` ``)
- Message: plain text
- Fix hint: arrow + backtick command (`` → `chmod 600 <path>` ``)

## Remediation section (from `_emit_tick_remediation()` in tick.sh)

```
### 🛠️ What to do

- `chmod 600 /var/lib/secmon/state.json` — Fix permissions
- `secmon --status` — Verify

▶ `chmod 600 /var/lib/secmon/state.json`
```

- H3 header with emoji
- Bullet list: `` `command` — description ``
- CTA: `▶ \`command\``

## CTA design rules

1. Show the **exact command** to run, not a Hermes slash command (`/secmon audit`)
2. Wrap commands in backticks so they're copy-paste ready on Telegram
3. Prepend with `▶` arrow for visual prominence
4. One primary action per notification

## `_stdout_remediation_hint()` per-alert-type design

| Source | Condition | Hint |
|--------|-----------|------|
| `self_protection` | permissions too open | `` → `chmod {expected} <path>` `` |
| `self_protection` | code file changed | `` → `git pull && sudo ./scripts/install.sh` `` |
| `self_protection` | scheduler missing | `` → `hermes cron list` `` |
| `self_protection` | symlink retargeted | `` → `ln -sf /path/to/trusted/checkout /opt/secmon` `` |
| `self_protection` | delivery target changed | `` → `grep deliver /etc/secmon/config.yaml` `` |
| `self_protection` | other | `` → `secmon --audit` `` |
| `fail2ban` / `brute_force` / `ssh_session` | any | `` → `fail2ban-client status sshd` `` |
| `outbound` / `c2` | any | `` → `ss -tlnp \| grep ESTAB` `` |
| `audit:*` | any | `` → `secmon --audit` `` |
| `botnet` | any | `` → `iptables -L BOTNET -n` `` |
| `anomaly` | any | `` → `secmon --check` `` |
| generic fallback | — | `` → `secmon --status` `` |

The hint should always be the **one command** that a human should run next — never "reply /secmon audit" chat jargon.

## Alert line -> remediation matching (tick.sh pattern matching)

When alert lines are Markdown-formatted (`**HIGH** \`self_protection\`: message`), the source name is wrapped in backticks. Bash patterns like `"self_protection:"` break because the colon follows a backtick, not the source name directly.

**Correct approach:** match on the **message content** (which is plain text), not the source-prefixed pattern:

```bash
# BROKEN — backtick before colon breaks the match:
if [[ "${out}" == *"self_protection: permissions too open"* ]]; then

# CORRECT — match on message body only:
if [[ "${out}" == *"permissions too open"* ]]; then
```

## Audit output format

```
## 🔍 Secmon Audit

### 📊 Overview

| Item | Value |
|------|------:|
| 🟡 **MEDIUM** | 1 |
| 🔵 **LOW / INFO** | 4 |
| **Risk Score** | 4 |
| **Total Findings** | 5 |

---

### 🟡 MEDIUM — 1 finding(s)

**1. ✏️ Modified persistence entry: systemd_timers**

> Existing persistence entry modified — potential hijack

`persist_modified`  •  🛡️ Threat Intel

     • Path: `systemd_timers`

---

### ℹ️ INFO — 4 finding(s)

**1. 👤 Top invalid users: ['admin', 'ubuntu', 'postgres', 'oracle', 'username']**

> SSH login attempts using non-existent usernames — recon activity

`invalid_users`  •  📝 Logs

**2. 🔄 2 persistent finding(s) from previous audit**

> Finding persists across multiple audit cycles — unresolved

`trend_persistent`  •  📈 Trends

---

▶ `secmon --audit`  —  Run full forensic audit

*🕐 2026-07-03 15:00 UTC*
```

- Generated by `format_audit_markdown()` in `output.py` — **NO raw JSON**
- Overview table uses generic headers (`Item | Value`) — shows only severity levels with findings (omits CRITICAL/HIGH rows when 0)
- Severity section headers show count: `### 🟡 MEDIUM — 1 finding(s)`
- **Every check_id has a human-readable one-liner** in blockquote via `CHECK_ID_EXPLANATIONS` dict — explains *why* it matters, not just *what* it is (e.g. "Existing persistence entry modified — potential hijack", "SSH login attempts using non-existent usernames — recon activity")
- Metadata inline: `` `check_id`  •  {Layer} `` on one line (no separate "Layer:" / "Check:" lines)
- Detail fields as indented bullet points (no JSON dicts)
  - `PID: 5397`, `Path: /tmp/...`, `Permissions: 0o644`, `Binary: /usr/bin/...`, `Expected: /usr/bin/sshd`, `Content: OnCalendar=daily`
- Layer numbers mapped to human names: 📁 File Integrity, 🌐 Network, ⚙️ Process, 🔑 Auth, 📝 Logs, 🛡️ Threat Intel, 📋 Compliance, 📈 Trends
- Empty findings → "✅ **No findings — system is clean.**"
- Cron wrapper (`audit.sh`) prints markdown directly — no ```json code fence; appends timestamp after output: `*🕐 2026-07-03 15:00 UTC*`
- **Trend-layer internal meta-checks** (`layer_count`, `trend_new`, `trend_resolved`, `trend_persistent`, `risk_increase`) are still emitted as INFO findings for transparency, but they are **filtered out of the before/after trend comparison** (resolved/persistent/new sets) via `_INTERNAL_TREND_CHECKS` so they don't pollute with meaningless "RESOLVED: layer_count" entries

## Daily digest format

```
## 📅 Secmon Daily Digest
*2026-07-02 08:00 UTC*

### 📊 24h Metrics

| Metric | Value |
|--------|------:|
| SSH Failures | 0 |
| Invalid SSH Users | 250 |
| Fail2ban Bans | 135 |

### 📈 Baseline Comparison

| Metric | Today | Baseline | Δ |
|--------|------:|---------:|---:|
| SSH Failures | 0 | 12 | -12 ✅ |
| Fail2ban Bans | 135 | 80 | +55 ⚠️ |

### 🔍 Summary

- **SSH failures:** 0
- **Invalid users:** 250
- **Fail2ban Bans:** 135
- **Unique attackers:** 77 IPs / 73 subnets
- **Findings:** 0

▶ `secmon --audit` — Full forensic audit
```

- `format_daily_digest()` in `output.py` generates this — no code fence around it
- Uses `METRIC_LABELS` dict for human-readable names (not raw underscore keys)
- Baseline table shows Today, Baseline, and Delta columns
- Large deltas (2 sigma+) get emoji hints: warning for above, checkmark for below
- Summary section shows key numbers at a glance with human labels
- Fallback when no baselines exist: italic "No baselines calibrated yet"

## Script deployment

Cron wrappers live at `~/.hermes/scripts/secmon/*.sh` as **copies** (not symlinks). Hermes cron resolves real paths and blocks scripts that resolve outside `~/.hermes/scripts/`.

After `git pull`, sync with:
```bash
sudo ./scripts/sync-cron.sh
```
